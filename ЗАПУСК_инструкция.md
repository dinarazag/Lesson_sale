## ЭТАП 1. Демо на Beget (сегодня)

### Шаг 1. Собрать сайт (на своём компьютере)

В Cursor открой терминал (Terminal → New Terminal) и выполни:

```
npm run build
```

Появится папка **`dist/`** — это и есть готовый сайт (набор файлов для загрузки).

> Если увидишь ошибку `Cannot find module @rollup/...` — выполни `rm -rf node_modules package-lock.json && npm install`, потом снова `npm run build`.

### Шаг 2. Загрузить на Beget (статический сайт)

Самый простой способ, VPS не нужен — обычный хостинг Beget:

1. В панели Beget создай сайт (домен или технический адрес вида `*.beget.tech`).
2. Открой **Файловый менеджер** Beget и зайди в папку сайта (обычно `public_html` или папка с именем домена).
3. Загрузи туда **всё содержимое папки `dist/`** (именно содержимое: `index.html`, папку `assets`, `.htaccess` и т.д. — а не саму папку `dist`).
   - Файл `.htaccess` уже лежит внутри `dist` после сборки — он нужен, чтобы страницы не «ломались» при обновлении. Убедись, что он загрузился (скрытые файлы, начинающиеся с точки, менеджер иногда прячет — включи показ скрытых).
4. Открой адрес сайта в браузере — должно открыться демо.

> Если весь проект нужно сначала выложить на GitHub — см. блок «Публикация на GitHub» ниже. Но для загрузки на Beget через файловый менеджер GitHub не обязателен.

### Альтернатива для айтишника: Beget VPS + Docker

Если знакомый предпочитает VPS — в проекте уже готов Docker:

```
docker compose up -d --build
```

Сайт поднимется на порту **8080** (меняется в `.env` → `APP_PORT`). SPA-маршрутизация и nginx уже настроены (`deploy/nginx.conf`). Подробности — в `DEPLOY.md`.

### Проверка

Открой сайт в **режиме инкогнито** (Cmd+Shift+N), чтобы увидеть путь как новый пользователь:
**лендинг → «Попробовать сейчас» → выбор роли и профиль → лента с уроками и «Забронировать».**
