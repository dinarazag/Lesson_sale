# Размещение на сервере

Инструкция для того, кто выкладывает сайт (хостинг-провайдер, администратор или фрилансер). Код писать не обязательно.

## Что получится после деплоя

- Сайт открывается в браузере по адресу сервера или домена.
- Данные пользователей пока хранятся **в браузере** (localStorage), общей базы на сервере нет — это ограничение текущей версии, не ошибка установки.

---

## Способ 1 — Docker (рекомендуется)

На сервере должны быть установлены **Docker** и **Docker Compose**.

1. Скопируйте проект на сервер (git clone или архив).
2. При необходимости настройте переменные:
   ```bash
   cp .env.example .env
   ```
3. Запустите:
   ```bash
   docker compose up -d --build
   ```
4. Сайт будет на порту **8080** (или на том, что указан в `.env` как `APP_PORT`).

Проверка: откройте `http://IP-СЕРВЕРА:8080` или `http://ваш-домен.ru:8080`.

Чтобы открывать сайт на стандартном порту 80 без `:8080`, настройте reverse proxy (nginx на хосте) или измените в `docker-compose.yml` строку портов на `"80:80"`.

Остановка: `docker compose down`  
Обновление после изменений в коде: `docker compose up -d --build`

---

## Способ 2 — VPS с nginx (без Docker)

1. На сервере: Node.js 20+ и nginx.
2. В папке проекта:
   ```bash
   npm ci
   npm run build
   ```
3. Скопируйте содержимое папки **`dist/`** в каталог на сервере, например `/var/www/lesson-sale/dist`.
4. Настройте nginx по примеру: [`deploy/nginx-vps.conf.example`](deploy/nginx-vps.conf.example).
5. Перезапустите nginx: `sudo nginx -t && sudo systemctl reload nginx`.

---

## Способ 3 — только статика (Netlify, Vercel, Timeweb и т.п.)

1. Соберите проект: `npm ci && npm run build`.
2. Укажите хостингу **корневую папку публикации**: `dist`.
3. Для SPA включите правило: **все пути → `index.html`** (на Netlify/Vercel это часто по умолчанию для Vite).

## Переменные окружения при сборке

| Переменная | Назначение |
|------------|------------|
| `VITE_APP_ID` | ID приложения (по умолчанию `local`) |
| `VITE_BASE44_APP_ID` | Для будущей интеграции с Base44 |
| `VITE_BASE44_FUNCTIONS_VERSION` | Версия функций Base44 |
| `VITE_BASE_PATH` | Если сайт в подпапке, например `/app/` |

Задаются в `.env` **до** `npm run build` или в `docker compose` через `build.args`.

---

## HTTPS и домен

- Домен указывается в DNS на IP сервера.
- На VPS с nginx: [Let's Encrypt](https://certbot.eff.org/) (`certbot --nginx`).
- В Docker часто ставят nginx/Caddy **перед** контейнером на порту 443.

---

## Когда понадобится программист

Чтобы у разных пользователей были **общие** данные, вход по паролю и оплата — нужен **бэкенд** и правки в `src/api/base44Client.js`. Текущий деплой выкладывает только интерфейс (фронтенд).
